Work 01: Because it's time, you'll make a line

Feb 10, 2022

Due: Tuesday 02/15 8:00am

Period 4 GitHub link: https://classroom.github.com/a/0aSrCPMb

Period 5 GitHub link: https://classroom.github.com/a/8nhNrTnX

You will write your own line algorithm!

  1. First decide what you will be programming in for this assignment. You can use any language you’d like, but you must write your own line algorithm (do not use any built in function that may exist for your language of choice).
    • I have provided frameworks for both c and python, these will be included when you create the assignment via GitHub classroom.
    • Also included at the room level is a .gitignore file that will prevent you from including image files in your submission. If your programming lanuage has temporary files (like .pyc files in python or .o files in c/c++), please add that to the .gitignore. The only files that should be pushed to GitHub are your makefile and any source code files.
    • Your makefile should like at the room level of your repository. I suggest this for your code as well, but some people do prefer using /src subdirectories, which is ok.
    • Remove all files that you are not using, so if you’re writing in python, get rid to the base_c directory and so on.
    • The provided source will also generate a test image. You can find this code in main.c or main.py, NO MATTER WHAT LANUAGE YOU USE your code must use the same inputs and generate the correct test image. A correct version of the image will be visible in the gallery.
  2. Code Bresenham’s line algorithm!
    • Start with octant 1.
    • Add the other octants one at a time, you’ll have a much easier time debugging doing that. (Trust me, I’ve done this a few times).
  3. Running make should result in displaying/saving an image that tests all octants, including slopes of 0, 1, -1 and undefined slopes. Use the tests already provided in main.c or main.py
  4. Don’t forget to upload an image to the gallery! - Follow all the submission guidelines at the top of this page.
    1. This image need not be the same as when your code runs.

If your are using my frameworks you only need to modify the following files:

  • c
    • draw_c
    • main_c
  • python
    • draw.py
    • main.py
