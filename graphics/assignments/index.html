Assignments

SUPER IMPORTANT SUMISSION GUIDELINES

  • You must include a makefile for each assignment!
    • Makefile help: https://www.stuycs.org/systems-dw/notes/#make
    • The makefile should invoke whatever is needed to compile and/or run your program.
    • When I type make, I expect to have your program generate and display an image.
      • For displaying, you have multiple options:
        • Use the built in display functions provided in display.py or display.c.
        • Have your makefile call the display command line utility after running your program.
        • Both of those options require a working imagemagick installation.
        • You could also use the open command line uility in your makefile (but not xdg-open).
    • If you cannot get the image to display (either programmatically or via the display command line program), then your program should print the name of the image file created.
  • The image created should thoroughly test the assignment by using the test code provided by me in main.c or main.py.
  • Every assignment (unless otherwise noted) will require both a github submission and an image gallery submission. Full credit will require both.
  • The gallery submission does not have to be the same as the test image generated when your code is tested.

We will be using GitHub Classroom to submit work. Each assignment with have a gh classroom link, clicking on it will create an assignment repository for you. This will work like any normal repository, so your github workflow should be the same. You just don’t need to worry about submitting your repository as a submodule are anything else.

  • For each assignment, I will provid frameworks for both c and python, these will be included when you create the assignment via GitHub classroom.
  • Also included at the root level is a .gitignore file that will prevent you from including image files in your submission. If your programming lanuage has temporary files (like .pyc files in python or .o files in c/c++), please add that to the .gitignore. The only files that should be pushed to GitHub are your makefile and any source code files.
  • Your makefile should like at the root level of your repository. I suggest this for your code as well, but some people do prefer using /src subdirectories, which is ok.
  • Remove all files that you are not using, so if you’re writing in python, get rid to the base_c directory and so on.

  • The provided source will also generate a test image. You can find this code in main.c or main.py, NO MATTER WHAT LANUAGE YOU USE your code must use the same inputs and generate the correct test image. A correct version of the image will be visible in the gallery.

  • NEVER EVER EVER add image files to github, adding image files will result in deductions <!–
    • All github submissions should be submodules in the correct period directory for the assignment. Please see the resources page for help on linking your own repository as a submodule.
      • You should maintain separate repositories for each assignment.
      • Submodule naming schema: lastF0f1
      • e.g. dyrland-weaverJon or konstantinovichSam or brooksPet
      • When working with a partner: lastF0f1f2_lastF0f1f2
      • e.g. brooksPet_dyrland-weaverJon
      • The first name should always be the earlier name alphabetically
      • NEVER EVER EVER add image files to github, adding image files will result in deductions
